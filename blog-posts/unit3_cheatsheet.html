Javascript Syntax
  • A prototype is a container for properties that are shared by all of its instances.
  • You can use Object.create(); to create an object with a specific prototype.
  • All objects have can have prototypes.
  • To find the prototype of an object use: Object.getPrototypeOf()
  • Every Javascript function has a prototype property by default and properties and methods can be attached onto it.
  • The prototype property is not enumberable, so it is not accessible in for/in loops.
  • Prototype chain provides the objects internal links to other objects.
  • Prototype attributes act as a characteristic of an object and tells us the objects "parent" or where the object inherited its properties from.
  • Use method overriding to override a prototypes inherited functions.
  • When an objects property is accessed and the interpreter can't find it, it follows the protolink and searches the prototype.
  • Prototype is important in JavaScript because JavaScript does not have classical inheritance based on Classes, therefore all inheritance in JavaScript is made possible through the prototype property.
  • Every object instance in JavaScript has a property named constructor, that points to its parent class.
